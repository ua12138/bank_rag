# 面试题库（interview_bank）

## 1. 项目总览类

### Q1：你这个 RAG 项目解决了什么业务问题？
- A：解决生产运维知识分散、检索效率低的问题，把手册、预案、工单统一入库，支持可追溯问答。
- 追问：为什么不用纯关键词搜索？
- 深挖点：稀疏检索与稠密检索在“术语命中”和“语义召回”上的互补关系。

### Q2：端到端链路是什么？
- A：文档解析 -> 清洗 -> 切块 -> 向量化 + BM25 -> 混合检索 -> RRF -> 可选重排 -> 生成答案。
- 追问：每一步最容易出问题的是哪一环？
- 深挖点：解析质量和检索质量如何影响最终幻觉率。

## 2. 检索与模型类

### Q3：为什么采用 BM25 + 向量混合检索？
- A：BM25 对告警码、表名、参数名这类关键词强；向量检索对口语化问题和语义改写更强，组合后召回更稳。
- 追问：融合为什么选 RRF？
- 深挖点：RRF 对不同打分尺度鲁棒，适合异构检索器融合。

### Q4：重排在链路中起什么作用？
- A：重排在候选集上做更细粒度相关性判断，提升 top-k 质量。
- 追问：为什么提供 `fast_mode` 跳过重排？
- 深挖点：延迟与精度的 trade-off。

### Q5：你们模型是怎么接入的？
- A：统一通过 SiliconFlow API 客户端接入 embedding/rerank/chat/vision。
- 追问：如何处理 embedding batch 限制？
- 深挖点：批量拆分与失败重试策略。

## 3. 多模态与数据治理类

### Q6：图片里的文字和表格是怎么识别的？
- A：链路是 RapidOCR -> pytesseract -> VLM 表格优先 -> VLM 通用 OCR -> CLIP 语义特征。
- 追问：CLIP 能直接识别表格内容吗？
- 深挖点：CLIP 与 OCR 在能力边界上的差异。

### Q7：为什么要把评估样例从 demo_kb 隔离？
- A：防止样例被切块入库污染检索结果，影响线上问答真实性。
- 追问：除了物理隔离，还做了什么防呆？
- 深挖点：seed 扫描白名单 + 排除目录/文件名策略。

## 4. 质量与评估类

### Q8：bad-case 的核心价值是什么？
- A：把“答错”变成可复现问题。通过 `retrieval_snapshot` 记录当时检索现场，定位是重写、召回、重排还是生成问题。
- 追问：bad-case 怎么回流到优化？
- 深挖点：bad-case -> ragas 数据集 -> 指标对比 -> 策略调整闭环。

### Q9：ragas 四个指标分别看什么？
- A：faithfulness（是否有证据支撑）、answer_relevancy（是否答到点上）、context_precision（召回噪声）、context_recall（证据覆盖）。
- 追问：如何设阈值？
- 深挖点：precision 与 recall 的平衡策略。

## 5. 性能与工程化类

### Q10：除了流式输出，你做了哪些性能优化？
- A：并行检索、查询缓存（TTL+LRU）、快速路径（fast_mode）。
- 追问：如何证明优化有效？
- 深挖点：对比 cache_hit 与 latency_ms 的 A/B 验证方法。

### Q11：多轮对话记忆怎么做，怎么控上限？
- A：按 `session_id` 存消息，上限由 `max_turns/max_chars` 控制，超限后做摘要压缩并清理历史。
- 追问：为何不无限保留上下文？
- 深挖点：成本、时延、噪声累积和错误传播。

## 6. 风险与改进类

### Q12：你认为当前方案最大风险是什么？
- A：评估链路是近似 ragas，需避免“指标绝对化”误用；另有编码可读性和 Milvus 集合治理问题。
- 追问：你下一步怎么改？
- 深挖点：官方 ragas 对照、回归自动化、运维规范固化。
